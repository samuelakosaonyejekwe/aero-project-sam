# Contributing

Thank you for your interest in this project.

This repository is a **personal case-study / portfolio project** by
**Samuel Akosa Onyejekwe** and is published under a proprietary
**All Rights Reserved** license (see [`LICENSE`](LICENSE)). It is made publicly
viewable for reference and demonstration purposes only.

Please read the following before getting in touch.

## What this means for contributions

- The codebase is **not open for unsolicited code contributions.** Forks, pull
  requests, and derivative works that reuse this project's code, data, figures,
  geometry/mesh files, or report are **not accepted without the Author's prior
  written permission.**
- You may **view** the repository and reference it for learning. Any other use —
  copying, executing, modifying, redistributing, or incorporating it into another
  work — requires written permission from the Author.

## Ways you can engage

Even though code contributions aren't accepted by default, the following are very
welcome:

- **Questions** about the methodology, the PyAnsys pipeline, or the results.
- **Bug reports** or corrections (e.g. an error in the analysis, a broken link,
  a typo in the report or README).
- **Suggestions** for improvements or follow-up studies.
- **Collaboration or reuse requests** — if you'd like to build on this work, get
  in touch to discuss terms.

### How to raise something

1. **Open an issue** on the repository describing the question, problem, or
   request. Please be specific and, for bugs, include enough detail to reproduce.
2. For **reuse, collaboration, or licensing** enquiries, contact the Author
   directly (see below).
